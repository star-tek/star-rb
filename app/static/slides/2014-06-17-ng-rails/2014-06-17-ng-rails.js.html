AngularJS + Rails = ng-rails

But first - Why Lineman?

“Lineman takes everything you love about building server-side applications so that you can find joy in your client-side applications”

IMO, javascript is good at doing javascript things. Grunt & Bower are great on the frontend. Source maps are nice too.

In Rails, if you look at the asset pipeline’s structure, you’ll notice how it mimics the app folder.

But really, those two concerns should be separated. IMO, Rails just isn’t great at managing frontend assets.

Using a single-page AngularJS app and API-only Rails. Lineman is perfect for separating these concerns.

Appistack has these loaded via gitmodules, which are easily substitued. I could sub in a Clojure API if I wanted to. Easy.

With the Rails-Lineman gem, they not 100% independent, but the project is more convenient this way. And much easier to deploy to Heroku.

The Rails API has Devise, Resque, Minitest & Capybara set up. It’s pretty vanilla. Rails serves the index page.

I had to make some changes to get the integration tests working. Set up helpers to update the links in test environment. Lineman expects you to use JS testing tools.

In LinemanJS, I’ve set up Angular/UI, Bower, UI-Bootstrap, Font Awesome. The App basically has a signup/login modals, a navbar widget and a few pages.

Frontend Problems I resolved:

  • Webfonts & Font Awesome
  • ngRoute/ui.router
  • Serving assets in test suite
  • Containerizing Lineman for multiple environments (apiUrl & CORS)
  • Compiling Jade templates with Lineman
